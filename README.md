# Scoreless : 시험 불안을 해소하는 AI 기반 시험 보강 시스템

> **"시험에 목숨 걸기 싫은 사람들을 위한, 과정 중심의 성적 보정 안전망"**
> 
> 단 한 두 번의 일회성 지필 시험으로 한 학기의 노력을 전부 평가받는 불공정함을 해결하기 위해, 평소의 꾸준한 학습 데이터를 기반으로 성적의 격차를 메워주는 **'학습 평가 완충 프로토타입(Learning Buffer System)'**입니다.

## 💡 기획 배경 (Why?)
특히 대학교에서 많은 학생들은 단 한 두 번의 시험으로 한 학기의 결과를 맞이하게 됩니다. 하지만 과도한 긴장감, 컨디션 난조가 겹치는 일회성 시험으로는 학생의 진짜 실력을 완전히 평가하기 어렵습니다. 

**"우리가 밤새 노력했던 과정이, 단 한 줄의 시험 점수로만 표현되어야 할까요?"**

저희가 진행한 유저 설문조사에서도 시험 자체를 100% 없애는 것에는 불안감을 느꼈지만, 시험 당일의 리스크를 평소 노력으로 보완하는 시스템의 필요성에는 깊이 공감했습니다. `ScoreLess`는 순간이 아닌 기간을, 휘발되는 단순 암기가 아닌 진짜 지식을 인정받을 수 있도록 **평소의 성실함을 점수로 증명하는 안전망**을 제안합니다.

## ✨ 핵심 기능 (What it does)

### 1. 결과가 아닌 '과정' 중심의 기록
사용자가 평소에 공부한 개념과 분야를 등록하고 학습 데이터를 누적해 나감으로써, 한 학기 동안 얼마나 지속적이고 성실하게 성장해 왔는지를 정량적으로 기록하고 타인에게 쉽게 어필할 수 있습니다.

### 2. 성취도를 한눈에 보는 '학습 대시보드'
등록된 모든 학습 주제와 과목들을 대시보드 형태로 한눈에 모니터링할 수 있으며, 지속적인 학습 관리를 돕습니다.

### 3. 기술적 차별점: AI 오남용 및 치팅(부정행위) 방지
생성형 AI의 무분별한 복사-붙여넣기나 대필을 통한 상시 평가 시스템의 악용을 막기 위해, 사용자의 평소 문장 입력 버릇, 오답 프로세스 등의 학습 패턴을 읽어내어 진짜 실력을 판별해 냅니다.

## 🛠️ 현재 구현 단계 (Current Milestone)
> 본 프로젝트는 아이디어톤 프로토타입 단계로, 핵심 UI 흐름과 로컬 데이터 저장 기능이 우선 구현되어 있습니다.

- **메뉴 네비게이션**: 좌측 상단의 "메뉴" 콤보박스를 통해 `학습 대시보드`와 `새 주제 등록` 화면을 자유롭게 전환할 수 있습니다.
- **새 주제 등록**: 사용자가 직접 학습 중인 개념 이름과 분야를 입력하고 등록하면, 실시간으로 대시보드에 새로운 카드가 추가됩니다.
- **주관적 자가 진단**: 아직 AI 상시 분석 기능이 탑재되기 전 단계이므로, 사용자가 대시보드에서 해당 과목의 '학습 종료 및 평가하기' 버튼을 눌러 본인의 이해도를 주관적으로 점검할 수 있도록 설계했습니다.
- **데이터베이스 연동**: 입력되고 변경되는 모든 학습 주제 정보는 SQLite3 데이터베이스(`scoreless.db`)에 안정적으로 저장됩니다.
- *※ 데이터 기반의 객관적 숙련도(CS) 계산 알고리즘 및 복습 주기 자동 생성 기능은 향후 AI 엔진 고도화와 함께 구현될 예정입니다.*

## 📂 파일 구조 (File Structure)
- **Frontend / UI**: Streamlit
- **Backend / Engine**: Python 3
- **Database**: SQLite3 (`scoreless.db`)

```text
Scoreless/
├── streamlit_app.py   # 메인 웹 어플리케이션 구동 및 UI 화면 구성
├── db_manager.py      # SQLite DB 연결 및 과목 데이터 추가/조회 관리
├── engine.py          # 학습 숙련도(CS) 알고리즘 및 코어 로직 담당
└── scoreless.db       # 사용자의 성실도 데이터가 저장되는 로컬 데이터베이스
```
